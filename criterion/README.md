# Criterion: AI-Powered Customer Support Quality Auditor

Criterion is an automated quality auditing platform that analyzes customer support interactions to measure agent performance, detect compliance violations, and provide real-time improvement recommendations.

## Project Structure
- `backend/`: Flask API and core analysis logic.
- `frontend/`: Vanilla JS dashboard and styling.
- `tests/`: Pytest suite for backend components.
- `sample_data/`: Sample chats and audio files for testing.
- `docs/`: Architecture and methodology documentation.

## Setup Instructions

### Prerequisites
- Python 3.10+
- Deepgram API Key (for audio transcription)
- OpenRouter API Key (for LLM scoring)

### Installation
1. Navigate to the `criterion` directory.
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Set up environment variables:
   ```bash
   cp .env.example .env
   # Add your API keys to .env
   ```

### Running the Application
1. Start the Flask backend:
   ```bash
   python -m backend.app
   ```
2. Open `frontend/index.html` in your browser. (You may need to serve it via a local web server if browser security blocks fetch requests to localhost).

## Key Features
- **Audio & Chat Processing**: Deepgram Nova-2 for transcription and sentiment.
- **QA Scoring Engine**: 8 core parameters across 4 weighted categories.
- **Prohibited Phrase Scanner**: Real-time regex detection for compliance.
- **Coaching Engine**: Actionable improvement tips generated by LLM.
- **Interactive Dashboard**: Radar charts, sentiment timelines, and leaderboards.

## Testing
Run the test suite:
```bash
pytest
```
